# UI 최종 수정사항

## 개요
현재는 사용자의 근태기록, 휴가 및 커스텀 휴일을 지정하고 달력에 표시하는 기능만 있다.
하지만 실제로 직장인들은 시간 외 수당을 받는 "연장근무(야근 등)", "휴일 근무" 등도 존재한다.
이러한 특근을 기입하고, 수당까지 계산해주는 기능을 만들고자 한다.
물론 특근은 FAB를 눌러 추가하며(근무에 특근 체크박스 등도 가능) 달력에도 표시되어야하고, 앱바나 기타 버튼 등을 눌러서 현황 및 수당계산까지도 확인 가능해야한다. 

## 모델링
- 기존에 구현된 entity와 model에서 수정이 필요한 부분을 수정한다.
- 혹은 아예 새로 추가해야한다면 추가한다.

## supabase 세팅
- 모델링을 기반으로 테이블 수정 및 추가 작업을 사용자에게 지시한다.

## Data 레이어 작성
- Calendar에 작성하는게 맞는지, 혹은 아예 다른 feature로 구분해야하는지 모르겠으므로 사용자에게 추천한다.

## 도메인 레이어 작성
- 앞서 작성한 데이터 레이어를 기반으로 Event, Repo, UseCase 등을 작성한다.

## UI 레이어 작성 
- Bloc을 수정한다. calendarBloc이 아니라면 새로 작성한다.
- UI작업을 진행한다. 특근 계산기 화면 등을 만들어야한다. 추천구조를 사용자와 논의한다.
- InfoPage에는 이제는 더이상 하나의 Card가 아니라, 일반근무와 특근까지 다 표기할 수있는 List가 보여야한다.

## 추가 사항
- 엣지케이스와 에러케이스를 잘 잡을 수 있도록 한다.

## 참고사항
- 특근은 근무시간 계산에서 제외한다. 
- 모든 구현은 단순하게, 복잡하지 않고 가독성 좋게 구현되어야한다.
- 추상체는 Repo만 만든다.
- 기존 디자인 시스템과 테마를 반드시 준수할 것
