## UI
### ThemeServie 추가사항
지금 각 typo별로 fontWeight를 w600 이런식으로 값을 박아넣고있는데,
ThemeService 자체에 그냥 semiBold, regular, light 이 세가지 weight를 선언해놓고 그걸 각 typo에 넣어준다.
외부에서도 copyWith을 활용해 fontWeight를 가져다 수 있도록 한다.

### CalendarPage
- 2026년 6월 이라고 적힌 앱바가 스크롤시 같이 내려가면 좋을 것 같다. 현재는 상단 고정이다.
- 달력 각 셀의 날짜 크기는 좀 줄여도 된다.
- 만약 근무시간이 10h20m 와 같이 길게. 표시되는 라벨이라면 셀에서 overflow에러가 발생한다. h,m을 없애던지 혹은 더 좋은 해결책이 있다면 생각해낸다.
- 일반근무와 연장근무 등의 dot이 함께 존재할 경우, cell에는 dot을 우선적으로 위에 배치하고 그 아래에 근무시간을 적는 방향으로 간다(column등 활용)
- 근태를 삭제하거나 기록하는 등의 행위를 하면, selectedDate가 오늘로 바뀌어버리는 현상

### InfoPage
- 근태Card의 앞에 primary색상으로 얇게 표시하는 container가 너무 옹졸하다. 아주 약간 크기를 더 늘려도 괜찮을 것 같다
- 공휴일일 경우, 날짜 밑에 해당 공휴일 이름이 빨간색으로 뜨는데, 이걸 없애자.

### WorkRecordBottomSheet
- 시간과 분을 각각 TextField에 입력할때, 옆 TextField를 누르면 키보드가 잠깐 내려갔다 올라온다. 이는 UX저하로 이어지므로, 해결책을 생각해본다.
- 키보드가 올라온 채로 CupertinoPicker를 띄우면 overflow 에러가 뜬다

### 전체적인 총평
- 특근현황 등의 페이지에서는 키보드 입력이 끝났는데도 키보드를 내릴 수 있는 방법이 존재하지 않는다.
- 전체적으로 overflow에 대한 처리를 진행해야할 것 같다.
- 앱바의 뒤로가기나 next,prev버튼 등의 버튼 hit범위와 인식률이 너무 낮다. 이는 매우 불편하다.
- 지금 UI코드가 매우 복잡하다. BLOC으로 뺄 수 있는 함수들은 모두 빼버리고, 불필요한 변수선언이나 복잡하게 구현되어있는 부분들을 싹 다 정리한다. (거시적으로 보지말고 feature 별로 page부터 시작해서 천천히 훑어본다)
- 컴포넌트화가 필요한 부분들도 찾아서 컴포넌트화를 진행할 수 있도록 한다.
